|고객|**김하늘 (21세, 음악대학 피아노 전공 지망생)**<br>- 고객 경험: 선천적 시각장애를 가지고 있으며, 음악대학 입시를 준비 중이다. 입시곡으로 지정된 곡들의 점자 악보가 거의 없어, 매번 시간을 들여 직접 손으로 익히고 암기한 뒤 연습한다. 하지만 연습 속도가 더디고, 잘못된 박자나 음정으로 외우면 교정하기가 어렵다. 입시 준비 과정에서 빠른 악보 확보와 정확한 연습 피드백이 절실하다.<br>- 고객 요구:<br>&nbsp;&nbsp;1. 빠른 점자 악보 변환: 일반 악보를 바로 점자 악보로 변환해, 입시 곡 연습 시간을 줄이고 싶다.<br>&nbsp;&nbsp;2. 실시간 연주 피드백: 내가 연주한 부분이 정확한지 즉시 알려주어, 틀린 습관이 자리 잡지 않도록 도와줬으면 한다.<br><br>**박은정 (42세, 음악 교육 기관 교사)**<br>- 고객 경험: 시각장애인 음악학교에서 여러 학생들을 지도하는 음악 교사다. 다양한 실력 수준과 음악적 요구를 가진 학생들을 동시에 관리해야 하지만, 점자 악보 제작에는 시간이 오래 걸리고 전문 인력이 부족하다. 따라서 모든 학생에게 맞춤형 자료를 제공하는 데 어려움을 겪고 있다.<br>- 고객 요구:<br>&nbsp;&nbsp;1. 자동 점자 변환: 일반 악보를 빠르게 점자 악보로 변환해, 다양한 곡을 즉시 수업에 활용하고 싶다.<br>&nbsp;&nbsp;2. 실시간 연주 피드백 지원: 학생들이 스스로 연습하면서 오류를 바로 잡을 수 있도록 AI 기반 피드백 기능을 제공받고 싶다.|
|Pain Point|- 점자 악보는 제작이 매우 어렵고, 수량이 적어 접근성이 극도로 제한적이다.<br>- 기존 점자 악보 제작은 시간과 비용이 많이 소요된다.<br>- 시각장애인은 악보를 손으로 읽고 외운 뒤 연주해야 하므로, 학습 효율이 매우 떨어진다.|
|사용할 소프트웨어 패키지의 명칭과 핵심기능/용도, 사용시나리오|**[Frontend]**<br>React Native : 모바일 앱 UI 개발. iOS/Android 환경에서 동일한 코드 기반으로 실행 가능하다.<br><br>**[Backend]**<br>Spring Boot : 백엔드 애플리케이션의 기본 프레임워크로, REST API 서버를 빠르고 안정적으로 구축한다.<br>Apache Kafka : 실시간 이벤트 스트리밍(연주 분석 결과 전달, 음성 명령 처리)을 지원한다.<br><br>**[협업 툴]**<br>Git/GitHub : 소스 코드 버전 관리 및 협업.<br>Figma : UI/UX 설계 및 프로토타입 제작.<br>Notion : 프로젝트 문서 관리 및 회의록 공유.<br>Discord : 팀 커뮤니케이션 및 실시간 협업.<br><br>**[AI / API]**<br>OMR(Optical Music Recognition) : 업로드된 악보 이미지를 MusicXML 등 기계가 읽을 수 있는 포맷으로 변환한다.<br>Braille Music Translation Engine: MusicXML 데이터를 국제 표준 점자 악보 규약에 맞춰 점자 악보로 변환한다.<br>TTS: 점자 악보 및 학습 가이드를 오디오로 읽어준다.<br>ASR(Automatic Speech Recognition): 사용자의 음성 명령을 인식하여 마디 이동, 반복 재생 등 기능을 수행한다.<br>Audio Analysis Model: 마이크로 입력된 연주를 분석해 박자·음정 오류를 탐지하고 피드백을 제공한다.|
